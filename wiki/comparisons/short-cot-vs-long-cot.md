# Short-CoT vs Long-CoT: Concise vs Extended Reasoning for Agent Training

A systematic comparison of two chain-of-thought paradigms for training web agents, revealed through [[webdancer-paper|WebDancer research]]: Short-CoT from instruction-following models (GPT-4o) vs Long-CoT from reasoning models (QwQ-Plus). The findings challenge conventional assumptions about reasoning depth and agent performance.

## Overview

When training autonomous web agents through [[two-stage-agent-training|supervised fine-tuning + reinforcement learning]], the quality and style of initial trajectories significantly impacts final performance. [[webdancer|WebDancer]] research reveals a surprising finding: shorter, more action-oriented reasoning (Short-CoT) from instruction models proves more effective for agent training than longer, more deliberative reasoning (Long-CoT) from reasoning models.

## Quick Comparison Table

| Aspect | Short-CoT (GPT-4o) | Long-CoT (QwQ-Plus) |
|--------|-------------------|---------------------|
| **Source Model Type** | Instruction-following | Reasoning model |
| **Average Actions** | 4.56 per trajectory | 2.31 per trajectory |
| **Average Tokens** | 510 per trajectory | 1599 per trajectory |
| **Reasoning Style** | Concise, task-focused | Extended, analytical |
| **Token/Action Ratio** | 112 tokens/action | 692 tokens/action |
| **Training Effectiveness** | High (better transfer) | Lower (poor transfer) |
| **RL Benefit** | Significant improvement | Limited improvement |
| **Inference Cost** | Lower (fewer tokens) | Higher (more tokens) |
| **Best For** | Agent training, action planning | Complex reasoning, deep analysis |

## Core Characteristics

### Short-CoT (GPT-4o Style)

**What It Is**:
- Chain-of-thought generated by instruction-following models
- Concise reasoning steps directly tied to actions
- Task-focused, goal-oriented thinking
- Practical action sequences with brief justifications

**Typical Trajectory Structure**:
```
Thought: I need to find information about X
Action: search("X")
Observation: [Search results...]
Thought: The first result looks most relevant
Action: visit(url1)
Observation: [Page content...]
Thought: I found the information, let me check one more source for confirmation
Action: search("X verification")
Observation: [More results...]
Thought: Now I have sufficient information to answer
Action: answer("Based on...")

Total: 4 actions, ~500 tokens
```

**Characteristics**:
- **Action density**: More actions per trajectory (4.56 avg)
- **Concise thoughts**: Brief reasoning between actions (~110 tokens per action)
- **Task focus**: Each thought directly supports next action
- **Efficient exploration**: More web interactions per trajectory
- **Clear sequences**: Easy to understand action flow

### Long-CoT (QwQ-Plus Style)

**What It Is**:
- Chain-of-thought generated by reasoning models (QwQ, o1, etc.)
- Extended deliberation before taking actions
- Deep analytical thinking with multiple angles
- Exploratory reasoning beyond immediate task

**Typical Trajectory Structure**:
```
Thought: I need to find information about X. Let me think carefully about what X means... X could refer to several concepts. In the context of this question, X likely means... I should consider multiple perspectives... The historical background of X is... The technical definition involves... Different researchers have different views... I need to search for authoritative sources... But which sources are most credible? Academic papers would be best... Let me formulate a precise search query...
Action: search("X")
Observation: [Search results...]
Thought: Interesting, I see several results. Let me carefully analyze each one... The first result is from... which has reputation... The abstract suggests... This aligns with my earlier reasoning about... However, I should be cautious because... The methodology appears... Let me visit the most promising source and evaluate its content thoroughly...
Action: visit(url1)
Observation: [Page content...]
Thought: Now I have the information. Let me synthesize... The key points are... Cross-referencing with my understanding... The evidence suggests... Therefore the answer is...
Action: answer("Based on...")

Total: 3 actions, ~1600 tokens
```

**Characteristics**:
- **Reasoning density**: Fewer actions per trajectory (2.31 avg)
- **Extended thoughts**: Long deliberation between actions (~690 tokens per action)
- **Analytical depth**: Multiple perspectives and considerations
- **Thorough exploration**: Deep analysis of each observation
- **Rich context**: Extensive background reasoning

## Performance in Agent Training

### Transfer Learning Effectiveness

**Short-CoT Training Data**:
- **High transfer**: Instruction models learn Short-CoT patterns easily
- **Action-oriented learning**: Clear action sequences provide strong training signal
- **Behavioral imitation**: Easy to replicate action-taking behavior
- **Cold-start success**: Essential for bootstrapping (vs. 5% direct RL)
- **RL compatibility**: Provides good initialization for RL refinement

**Long-CoT Training Data**:
- **Poor transfer**: Reasoning model knowledge doesn't transfer well to instruction models
- **Reasoning vs action gap**: Long deliberation doesn't teach action-taking
- **Sparse action signal**: Fewer actions mean less behavioral guidance
- **Limited RL benefit**: Reasoning models benefit less from RL in agentic tasks
- **Verbosity overhead**: Token cost without proportional performance gain

**Key Finding from WebDancer**: Short-CoT trajectories (7,678 samples) + Long-CoT trajectories (6,550 samples) combined training, but Short-CoT provides majority of learning signal.

### Supervised Fine-Tuning (SFT) Stage

**Short-CoT for SFT**:
- **Clear action patterns**: Model learns when to search vs visit vs answer
- **Efficient sequences**: Learns to take multiple focused actions
- **Task completion**: Understands how to navigate toward goals
- **Practical reasoning**: Learns just enough thinking to guide actions
- **Strong initialization**: Prepares model well for RL stage

**Long-CoT for SFT**:
- **Deliberation patterns**: Model learns extensive reasoning style
- **Fewer actions**: May learn to "think more, act less"
- **Analysis over execution**: Bias toward deliberation rather than action
- **Transfer challenges**: Reasoning models' thought patterns difficult to imitate
- **Initialization mismatch**: May not align with RL optimization objectives

**Trade-off**: Short-CoT teaches "how to act," Long-CoT teaches "how to think deeply" — for agents, action-taking is more critical.

### Reinforcement Learning (RL) Stage

**After Short-CoT Initialization**:
- **Strong foundation**: Already knows basic action sequences
- **RL refinement**: Can focus on optimizing action selection
- **Exploration improvement**: RL extends action repertoire
- **Generalization gains**: RL helps adapt to out-of-distribution questions
- **Significant improvement**: Large gain from RL fine-tuning

**After Long-CoT Initialization**:
- **Weaker foundation**: Fewer action patterns internalized
- **RL struggles**: Sparse action sequences harder to optimize
- **Limited exploration**: Less action diversity to build upon
- **Reasoning models limitation**: Reasoning models benefit less from RL in agentic tasks (WebDancer finding)
- **Marginal improvement**: Smaller gain from RL fine-tuning

**Sparse Reward Challenge**:
- RL uses binary correctness + answer quality (sparse signal)
- More actions = more gradient signal for learning
- Short-CoT's 4.56 actions provide ~2x learning signal vs Long-CoT's 2.31 actions
- Sparse rewards insufficient for optimizing Long-CoT reasoning depth

## When Each Excels

### Short-CoT Excels For

**Agent Training** (Primary Use Case):
- Training autonomous web agents
- Teaching action-selection policies
- Building behavioral repertoires
- Cold-start initialization for RL
- Cost-efficient training data generation

**Action-Oriented Tasks**:
- Web navigation and information seeking
- Tool use and API interaction
- Multi-step task execution
- Goal-directed behavior
- Practical problem-solving

**Resource-Constrained Scenarios**:
- Lower inference cost (510 vs 1599 tokens)
- Faster execution (more actions, less deliberation)
- Scalable deployment (lower token consumption)
- Real-time interaction (quicker responses)

**Example Use Case**: Training [[webdancer|WebDancer]]-style research agents that need to efficiently navigate web environments and synthesize information.

### Long-CoT Excels For

**Complex Reasoning Tasks** (Non-Agentic):
- Mathematical problem-solving
- Logical puzzles and proofs
- Scientific reasoning
- Multi-step analytical tasks
- Problems requiring deep deliberation

**Reasoning Model Evaluation**:
- Demonstrating model's reasoning capabilities
- Transparency in decision-making process
- Educational content (showing thought process)
- Debugging complex reasoning chains

**When Actions Are Expensive or Risky**:
- Situations where thinking is cheaper than acting
- High-cost actions requiring careful planning
- Irreversible decisions needing thorough analysis
- Safety-critical scenarios requiring validation

**Example Use Case**: Using QwQ-Plus or o1 for complex mathematical proofs where each reasoning step needs careful validation before proceeding.

### Hybrid Approaches

**Combining Both**:
- **Short-CoT for navigation**: Use concise reasoning for web interaction
- **Long-CoT for synthesis**: Use deep reasoning for final answer generation
- **Adaptive reasoning**: Short for simple tasks, long for complex
- **Training mix**: Both types in training data (WebDancer approach)

**Potential Benefits**:
- Best of both worlds: Action efficiency + reasoning depth
- Task-appropriate reasoning depth
- Balanced token cost and performance

**Open Question**: How to automatically determine when to use which style?

## Trade-offs Analysis

### Efficiency vs Depth

**Short-CoT**:
- **Efficient**: 112 tokens per action
- **Sufficient**: Provides enough reasoning to guide actions
- **Practical**: Optimizes for task completion
- **Cost-effective**: Lower inference and training costs
- **Trade-off**: May miss subtle considerations

**Long-CoT**:
- **Thorough**: 692 tokens per action
- **Comprehensive**: Considers multiple perspectives
- **Analytical**: Deep exploration of reasoning space
- **Expensive**: Higher costs for marginal agent benefit
- **Trade-off**: Extensive reasoning doesn't improve action selection proportionally

### Action Density vs Reasoning Density

**Short-CoT**:
- **More actions** (4.56 avg): More web interactions per trajectory
- **More exploration**: Visits more sources, searches more queries
- **Better training signal**: Each action provides learning opportunity
- **Practical exploration**: Trial-and-error learning style
- **Trade-off**: May take unnecessary actions without thorough planning

**Long-CoT**:
- **Fewer actions** (2.31 avg): Less web interaction per trajectory
- **More deliberation**: Extensive thinking before each action
- **Careful selection**: Each action carefully considered
- **Analytical approach**: Plan thoroughly, act sparingly
- **Trade-off**: May overthink simple actions, miss exploration opportunities

### Transfer Learning vs Direct Capability

**Short-CoT**:
- **High transfer**: Instruction models easily learn Short-CoT patterns
- **Behavioral learning**: Action sequences well-suited for imitation
- **RL synergy**: Provides good foundation for RL refinement
- **Generalizable**: Patterns transfer across tasks
- **Trade-off**: Limited reasoning depth in source data

**Long-CoT**:
- **Poor transfer**: Reasoning models' capabilities don't transfer to instruction models
- **Knowledge gap**: Deep reasoning style difficult to imitate
- **Limited RL benefit**: Sparse actions harder to optimize
- **Capability ceiling**: Instruction models may not reach reasoning model depth
- **Trade-off**: Superior source capability doesn't guarantee superior agent training data

## Model Type Considerations

### Instruction Models (GPT-4o, Claude-3.5, etc.)

**Characteristics**:
- Optimized for following instructions and task completion
- Concise, goal-oriented responses
- Practical reasoning sufficient for action selection
- Native Short-CoT generation

**For Agent Training**:
- **Generating Short-CoT**: Natural fit, generates efficient action sequences
- **Learning from Short-CoT**: High compatibility, easy transfer
- **Learning from Long-CoT**: Poor compatibility, reasoning style mismatch
- **RL optimization**: Responds well to RL fine-tuning

**Recommendation**: Use instruction models to generate training data for instruction-model agents.

### Reasoning Models (QwQ-Plus, o1, etc.)

**Characteristics**:
- Optimized for complex reasoning and deliberation
- Extended chain-of-thought with multiple perspectives
- Deep analytical thinking before conclusions
- Native Long-CoT generation

**For Agent Training**:
- **Generating Long-CoT**: Natural fit, generates thorough reasoning
- **Learning from Short-CoT**: Mixed compatibility, may prefer deeper reasoning
- **Learning from Long-CoT**: High compatibility for reasoning, low for actions
- **RL optimization**: Limited benefit in agentic tasks (WebDancer finding)

**Recommendation**: Use reasoning models for complex reasoning tasks, not agent training data generation (unless training another reasoning model).

### Cross-Model Transfer Challenge

**The Gap**:
- Instruction models cannot easily learn reasoning models' deliberation style
- Reasoning depth doesn't transfer to action-taking capability
- Different optimization objectives (instruction-following vs reasoning)
- Architectural differences may limit transfer

**Implication**: Match training data source model type to target model type for best results.

## Empirical Evidence from WebDancer

### Training Data Composition

**Short-CoT Trajectories**:
- 7,678 trajectories from GPT-4o
- 4.56 actions per trajectory avg
- 510 tokens per trajectory avg
- High-quality action sequences

**Long-CoT Trajectories**:
- 6,550 trajectories from QwQ-Plus
- 2.31 actions per trajectory avg
- 1599 tokens per trajectory avg
- Deep reasoning content

**Total**: 14,228 trajectories for cold-start SFT

### Performance Results

**After SFT + RL Training**:
- **GAIA Benchmark**: 51.5% average accuracy
  - Level 1 (Easy): 61.5%
  - Level 2 (Medium): 50.0%
  - Level 3 (Hard): 25.0%
- **WebWalkerQA Benchmark**: 47.9% average accuracy
  - Easy: 52.5%
  - Medium: 59.6%
  - Hard: 35.4%

**Key Observations**:
- Strong performance achieved despite Long-CoT transfer challenges
- Short-CoT likely provides majority of learning signal
- RL refinement improves upon SFT initialization
- Action-oriented training data proves effective

### Ablation Insights (Implicit)

**Direct RL without SFT**: Only 5% success rate
- Demonstrates necessity of cold-start trajectories
- Sparse rewards insufficient alone
- Both Short-CoT and Long-CoT needed for initialization

**SFT + RL vs SFT alone**: Significant improvement from RL
- RL stage adds generalization capability
- On-policy learning improves action selection
- Short-CoT foundation enables effective RL refinement

**Implicit Finding**: Short-CoT's higher action density likely provides stronger training signal, though paper doesn't explicitly ablate Short-only vs Long-only vs Combined.

## Practical Recommendations

### For Agent Training

**Use Short-CoT When**:
- Training web agents for navigation and information seeking
- Building action-oriented policies
- Working with instruction-following models
- Optimizing for inference efficiency
- Planning RL fine-tuning after SFT

**Best Practice**:
- Generate trajectories using instruction models (GPT-4o, Claude-3.5)
- Focus on action sequences and practical reasoning
- Aim for 4-6 actions per trajectory
- Keep reasoning concise (100-200 tokens per action)
- Ensure clear thought-action-observation structure

### For Complex Reasoning

**Use Long-CoT When**:
- Tasks require deep analytical thinking
- Mathematical or logical reasoning is primary goal
- Actions are expensive or irreversible
- Demonstrating reasoning process is valuable
- Working with reasoning models as target

**Best Practice**:
- Generate trajectories using reasoning models (QwQ-Plus, o1)
- Allow extensive deliberation before actions
- Aim for thorough analysis (500-1000 tokens per reasoning step)
- Prioritize reasoning quality over action efficiency
- Validate reasoning steps carefully

### For Hybrid Systems

**Combine Both When**:
- Training data includes diverse reasoning depths
- Task distribution varies in complexity
- Want model to adaptively choose reasoning depth
- Maximizing training data diversity

**Best Practice** (WebDancer Approach):
- Generate both Short-CoT and Long-CoT trajectories
- Filter for quality (multi-stage filtering)
- Combine in training dataset
- Let model learn from both styles
- Expect Short-CoT to dominate learning signal for action-taking

## Future Directions

### Open Research Questions

**For Short-CoT**:
- Can we make Short-CoT slightly deeper without losing action efficiency?
- What is optimal token/action ratio for agent training?
- How to balance conciseness with reasoning quality?

**For Long-CoT**:
- Can we improve Long-CoT transfer to instruction models?
- Is there a way to extract action-relevant reasoning from Long-CoT?
- Can distillation bridge reasoning model → instruction model gap?

**For Reasoning Depth Adaptation**:
- Can models learn to dynamically choose reasoning depth?
- What cues indicate when deep reasoning is beneficial?
- How to train models to be efficiently adaptive?

### Potential Improvements

**Enhanced Short-CoT**:
- Slightly richer reasoning while maintaining action density
- Better search query formulation
- More explicit source evaluation
- Structured reasoning templates

**Distilled Long-CoT**:
- Extract key reasoning from Long-CoT, compress to Short-CoT length
- Keep action sequences, summarize deliberation
- Combine Long-CoT insights with Short-CoT efficiency
- Best of both worlds through compression

**Adaptive Reasoning**:
- Train models to recognize task complexity
- Simple tasks → Short-CoT efficiency
- Complex tasks → Long-CoT depth
- Dynamic adjustment based on observations

## Conclusion

### Clear Winner for Agent Training: Short-CoT

**Evidence**:
- Higher action density (4.56 vs 2.31) provides more training signal
- Better transfer to instruction models
- More effective RL foundation
- Lower inference cost (510 vs 1599 tokens)
- WebDancer's success demonstrates effectiveness

**Key Insight**: For training autonomous agents, action-oriented concise reasoning outperforms deep deliberative reasoning, despite the latter's apparent sophistication.

### Context-Dependent Value

**Not Universal**:
- Short-CoT for agents, Long-CoT for reasoning
- Model type matters: match source to target
- Task characteristics determine optimal style
- Resource constraints influence choice

### Surprising Finding

**Counterintuitive Result**:
- More reasoning ≠ better agent training
- Reasoning models' knowledge transfers poorly to instruction models
- Action density more important than reasoning depth
- Efficiency beats thoroughness for agentic tasks

**Implications**: Challenge assumptions about reasoning depth and agent capability; focus on action-taking behavior rather than deliberation.

### Practical Takeaway

**For Practitioners**: When training web agents or similar action-oriented systems, prioritize Short-CoT trajectories from instruction models over Long-CoT trajectories from reasoning models, even though the latter appear more "intelligent."
